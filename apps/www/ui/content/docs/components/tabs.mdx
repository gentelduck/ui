

## API Reference

### Components

#### **Tabs**
The root component that manages the tab navigation logic and state.

- **Props**: Inherits all properties from `TabsPrimitive.Root`.

---

#### **TabsList**
A container for tab triggers, designed to hold the clickable tab elements.

- **Props**: Inherits all properties from `TabsPrimitive.List`.

| **Name**     | **Type**  | **Default** | **Description**                                                            |
|--------------|-----------|-------------|----------------------------------------------------------------------------|
| `className`  | `string`  | `undefined` | Additional CSS classes for styling the list container.                     |

- **Default Styles**:  
  - Rounded container with muted background.
  - Inline-flex layout with spacing and text-muted foreground.

---

#### **TabsTrigger**
A clickable element that acts as a trigger for its corresponding tab content.

- **Props**: Inherits all properties from `TabsPrimitive.Trigger`.

| **Name**     | **Type**  | **Default** | **Description**                                                            |
|--------------|-----------|-------------|----------------------------------------------------------------------------|
| `className`  | `string`  | `undefined` | Additional CSS classes for styling the tab trigger.                        |

- **Default Styles**:  
  - Responsive design with padding, transition effects, and focus-visible rings.  
  - Supports an active state with distinct background and foreground styles.

---

#### **TabsContent**
A container for the content of a specific tab. It is displayed when its corresponding trigger is active.

- **Props**: Inherits all properties from `TabsPrimitive.Content`.

| **Name**     | **Type**  | **Default** | **Description**                                                            |
|--------------|-----------|-------------|----------------------------------------------------------------------------|
| `className`  | `string`  | `undefined` | Additional CSS classes for styling the content area.                       |

- **Default Styles**:  
  - Margin-top spacing and focus-visible ring for accessibility.

---

### Accessibility
The `Tabs` system is fully accessible, adhering to WAI-ARIA guidelines. It supports keyboard navigation, focus management, and appropriate ARIA roles for triggers and content.

---

### Customization

All components in the `Tabs` system support the `className` prop, allowing you to override default styles with utility classes or custom styles. Additionally, the `cn` utility ensures conditional class merging for enhanced customization.
